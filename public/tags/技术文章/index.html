聊聊 nostr 和 审查

聊聊 nostr 和 审查 引子：一场“下架”引发的集体追问 2023 年初，一款名为 Damus 的 iOS 社交应用在苹果 App Store 上线，仅一天后便从中国区商店悄然消失。官方公告语焉不详，仅称“因违反中国法律法规”，但用户社区迅速达成共识：这并非偶然的技术合规问题，而是一次对新型去中心化通信协议——nostr（Notes and Other Stuff Relayed）——的首次现实尺度审查响应。 有趣的是，Damus 本身并不托管任何内容；它不运营服务器，不存储用户消息，甚至不验证用户身份。它只是一个轻量客户端，连接到全球数百个公开、自愿运行的中继服务器（relays），通过私钥签名发布加密消息，并订阅他人广播的事件。它的核心逻辑完全运行在用户设备上，数据流经第三方中继，但中继本身无权解密、编辑或删除内容——除非主动选择不转发。 这种架构让 Damus 成为首个在主流移动平台大规模落地的、真正意义上“抗审查”的社交客户端。而它的下架，恰恰印证了其设计目标：不是规避审查，而是让审查在技术上变得低效、昂贵且不可持续。 本文将深入拆解 nostr 协议的本质机制，厘清它与传统中心化社交平台（如微博、Twitter/X）、乃至其他去中心化方案（如 Mastodon、ActivityPub）的根本差异；我们将逐层剖析其密码学基础、事件模型、中继协作逻辑、客户端行为规范；并通过可运行的 Python 与 JavaScript 示例，亲手构建一个最小可行的发布-订阅流程；最后，我们将直面最尖锐的问题：nostr 真的无法被审查吗？它的“抗审查性”是否存在边界？当国家力量、商业平台、网络基础设施与协议层发生碰撞时，技术理想主义能否守住底线？这场关于“谁控制消息、谁定义真相、谁承担后果”的博弈，远未结束——它刚刚进入第一回合。 本解读不预设立场，但坚持一个前提：理解技术，是参与公共讨论的第一步。只有看清协议如何工作，我们才能理性判断——它究竟在抵抗什么，又可能纵容什么。 一、协议溯源：nostr 不是“另一个 Twitter”，而是一张签名消息的广播网 要理解 nostr 的颠覆性，必须先放下“社交网络”的旧有范式。它既不是微博那样的信息流平台，也不是微信那样的熟人关系链工具，更不是 Mastodon 那样基于服务器联邦的分布式微博客系统。nostr 的原始定位极为朴素：一个极简的、基于公钥密码学的、不可篡改的消息广播协议。 它的诞生背景极具象征意义：2020 年底，Twitter 创始人 Jack Dorsey 在推特上发起一项实验——“如果我们要重建一个开放的社交网络，它应该有多小？” 他提出一个核心思想：身份即公钥，消息即签名，分发靠中继，验证靠数学。这一想法由开发者 Globo 迅速实现为一份仅 300 行的原始规范草案，随后演变为今日的 nostr-protocol 开源标准。 1.1 核心哲学：三原则定义协议边界 nostr 严格遵循三条设计铁律，它们共同划定了协议的能力边界与安全假设： 身份即公钥（Identity is Public Key） 用户没有用户名、邮箱、手机号或任何中心化注册凭证。唯一标识是一个 32 字节的比特币风格 Secp256k1 公钥（以 npub 编码为 Base32 字符串）。私钥永远离线保存于用户设备，所有操作均由私钥签名完成。这意味着：创建账户零成本、零信任、零依赖；注销账户即销毁私钥——彻底、不可逆、无需通知任何人。...
